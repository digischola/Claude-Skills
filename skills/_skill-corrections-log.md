# Skill Corrections Log

One file across ALL skills. Append-only. Every time a skill is patched mid-run because something was wrong, one line gets logged here. No per-skill ceremony, no separate wiki folder, no hooks needed.

## Purpose

Mayank doesn't have a way to know if skills are getting better over time. The SKILL.md "Learnings & Rules" section is per-skill and gets pruned. This file is the cross-skill pattern memory.

After a few weeks of entries, patterns become visible: "scope-creep happened 3 times → audit other skill boundaries before they break the same way." That's the only point of this file. If it doesn't pay off after 8 weeks of use, delete it.

## Rules for entries

- **Who writes:** Claude, automatically, as part of any session that patches a skill.
- **When:** the moment a skill correction is applied (per the 7+3 rule). Same pass, no separate step.
- **Format:** one line per correction. Date | skill | mistake-type | one-line summary | files-touched-count.
- **Mistake types** (use these tags so patterns aggregate):
  - `scope-creep` — skill bled into next skill's job
  - `mode-confusion` — skill mode (FRESH/UPDATE/etc) misdefined or misapplied
  - `validator-gap` — validator missed a real failure
  - `validator-false-positive` — validator flagged a non-issue
  - `prompt-drift` — generated prompt produced wrong output type
  - `deliverable-gap` — promised output not produced
  - `path-detection` — skill couldn't find a file/folder it needed
  - `output-format` — output structure broke downstream consumer
  - `voice-rule` — generated content violated voice rules
  - `other` — doesn't fit (use sparingly; if used 3x, add a new tag)
- **What NOT to log:** content edits to client deliverables (those go in client wiki/log.md), one-off bugfixes that aren't pattern candidates, typo fixes.

## Pattern review (manual, monthly)

Once a month — or whenever it feels useful — Mayank or Claude reads this file end-to-end and asks:
1. Which mistake type has the most entries? That's the pattern to harden against next.
2. Which skills appear most? Those need the most maintenance.
3. Are entries thinning out per month? That's the improvement signal.

No dashboard, no metrics file, just read it.
