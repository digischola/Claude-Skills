# Copywriting Frameworks Reference

Load this file in Step 3 of the ad-copywriter skill. Contains framework definitions, selection logic, and proof integration patterns.

## Framework Selection Logic

| Campaign Type | Platform | Framework | Why |
|---|---|---|---|
| Brand Search RSA | Google | Feature-Benefit-Proof | Searcher knows brand — reinforce with proof |
| Non-Brand Search RSA | Google | Feature-Benefit-Proof | Searcher has intent — match feature to need |
| Shopping/PMax | Google | Feature-Benefit-Proof | Product-focused, spec-driven |
| Prospecting (problem-solving) | Meta | PAS | Interrupt with problem recognition |
| Prospecting (transformation/lifestyle) | Meta | BAB | Paint the before/after vision |
| Retargeting | Meta | Social Proof Stack | They've seen you — now prove others trust you |
| Video ads (any platform) | Both | AIDA | Sequential storytelling suits video frames |
| Long-form primary text (Meta) | Meta | PAS or AIDA | More space = can develop the argument |

**Decision shortcut:** If unsure between PAS and BAB for Meta prospecting, ask: "Does the product solve a pain point (PAS) or enable a transformation (BAB)?" Pain = PAS. Aspiration = BAB.

## Framework 1: PAS (Problem-Agitate-Solution)

**Best for:** Products that solve a specific, felt problem. Meta prospecting for service businesses, SaaS, health/wellness with a pain angle.

**Structure:**
1. **Problem** — Name the pain the persona experiences. Be specific, not generic.
2. **Agitate** — Amplify the consequence of NOT solving it. Emotional weight.
3. **Solution** — Present the product as the resolution. Include proof.

**Headline formula:** [Problem keyword] + [Solution hint]
**Primary text formula:** [Problem sentence] → [Agitate sentence] → [Solution + proof sentence] → [CTA]

### PAS Examples

**Wellness retreat (pain angle):**
- Problem: "Still running on empty after your last 'vacation'?"
- Agitate: "Resort pools and buffets don't fix burnout. You come back just as depleted."
- Solution: "7-day guided yoga retreat in Rishikesh. 94% of guests report lasting calm 30 days later. Book your October dates."

**SaaS project management:**
- Problem: "Losing hours to status update meetings?"
- Agitate: "Your team spends 5+ hrs/week reporting instead of building."
- Solution: "One dashboard. Every project. $49/user. Try free for 14 days."

## Framework 2: BAB (Before-After-Bridge)

**Best for:** Transformation products — retreats, coaching, luxury, lifestyle brands. Meta prospecting where the emotional gap between current state and desired state is large.

**Structure:**
1. **Before** — Describe the persona's current (undesirable) state. Empathy, not judgment.
2. **After** — Describe the transformed state. Sensory, specific, aspirational.
3. **Bridge** — The product is the bridge. How to get from Before to After.

**Headline formula:** [After-state keyword] + [Bridge hint]
**Primary text formula:** [Before state] → [After state] → [Bridge = product + proof] → [CTA]

### BAB Examples

**Luxury wellness retreat:**
- Before: "You're juggling 60-hour weeks, screen fatigue, and sleep that never feels like enough."
- After: "Imagine waking to birdsong, yoga at sunrise, three weeks of calm packed into seven days."
- Bridge: "Ananda Wellness Retreat. Voted 'Best Yoga Retreat NSW' by Wellness Australia. Oct-Mar bookings open."

**Architecture/design product:**
- Before: "Your backyard is wasted space — too small for a guest house, too big to ignore."
- After: "A Dezeen-featured studio that earns $74K/year in rental income."
- Bridge: "The Oculus by Retreat House. Master Builder certified. Starting from $210K. Book a site assessment."

## Framework 3: Feature-Benefit-Proof (FBP)

**Best for:** All Google RSAs. Search intent is high — match feature to the searcher's need and prove credibility in 30-character headlines.

**Structure:**
1. **Feature** — What the product has/does (concrete, specific)
2. **Benefit** — What that means for the buyer (outcome, saving, gain)
3. **Proof** — Why they should believe you (certification, stat, media mention)

**RSA application:** Headlines rotate freely, so each headline must work standalone. Mix feature headlines, benefit headlines, and proof headlines across H1-H15.

### FBP Headline Patterns (each ≤ 30 chars)

| Type | Pattern | Example |
|---|---|---|
| Feature | [Specific feature] | "15 Yoga Sessions Included" |
| Benefit | [Outcome verb] + [result] | "Save 5 Hours Every Week" |
| Proof | [Authority] + [credential] | "Dezeen Featured Design" |
| Price | [Price point] + [qualifier] | "From $49/User/Month" |
| CTA | [Action verb] + [object] | "Book Your Free Consult" |
| Urgency | [Time constraint] | "Oct-Mar Dates Filling Fast" |
| Brand | [Brand name] + [descriptor] | "Retreat House | Tiny Homes" |

### FBP Description Patterns (each ≤ 90 chars)

| Type | Pattern |
|---|---|
| Feature-Benefit | "[Feature]. [Benefit in outcome terms]. [CTA]." |
| Proof-Benefit | "[Credential/stat]. [What that means for buyer]. [CTA]." |
| Problem-Solution | "[Pain point]? [Product solves it]. [Proof]. [CTA]." |

## Framework 4: AIDA (Attention-Interest-Desire-Action)

**Best for:** Video scripts, long-form Meta primary text. Sequential format suits time-based media.

**Structure:**
1. **Attention** — Hook in first 1-3 seconds (video) or first line (text). Pattern interrupt.
2. **Interest** — Develop the story. Features, context, relevance to the persona.
3. **Desire** — Emotional payoff. Social proof, transformation visualization, scarcity.
4. **Action** — Clear, single CTA. Tell them exactly what to do next.

### AIDA Video Application

| Frame | AIDA Stage | Duration (15s ad) | Duration (30s ad) |
|---|---|---|---|
| Frame 1 | Attention (Hook) | 0-3s | 0-5s |
| Frame 2 | Interest | 3-6s | 5-12s |
| Frame 3 | Interest/Desire | 6-10s | 12-20s |
| Frame 4 | Desire | 10-13s | 20-25s |
| Frame 5 | Action (CTA) | 13-15s | 25-30s |

### AIDA Text Application (Meta Primary Text)

Line 1: [Hook — stop the scroll, pattern interrupt]
Line 2-3: [Interest — relevant detail, specifics]
Line 4: [Desire — proof, transformation, social validation]
Line 5: [Action — CTA with urgency or incentive]

## Framework 5: Social Proof Stack

**Best for:** Meta retargeting. Audience already knows the brand — now overcome objections with evidence.

**Structure:** Stack 2-3 proof elements in descending authority:
1. **Authority proof** — media mention, certification, industry award
2. **Statistical proof** — numbers, percentages, measurable outcomes
3. **Social proof** — customer testimonials, review count, user count
4. **CTA** — reminder CTA (they've already shown interest)

### Social Proof Stack Patterns

**Pattern A — Media + Stat:**
"Featured in [Publication]. [Stat that matters]. [CTA]."

**Pattern B — Testimonial + Authority:**
"'[Short quote]' — [Customer name]. [Certification/award]. [CTA]."

**Pattern C — Numbers Stack:**
"[User count] + [satisfaction %] + [specific outcome stat]. [CTA]."

## Proof Integration Patterns

How to weave proof elements from the creative brief's `proof_elements` array into copy without forcing them.

### Rules
1. **Match proof to persona.** Design buyers care about Dezeen. Budget-conscious buyers care about ROI stats. Certification matters to risk-averse buyers.
2. **One proof per copy block.** Stacking 3 proofs in one headline feels desperate. Spread across variants.
3. **Proof as headline vs proof in description.** Short proofs (≤ 30 chars) → headline. Long proofs → description.
4. **Natural integration.** Proof should answer "why should I believe this?" — not feel bolted on.

### Proof Type → Copy Position

| Proof Type | Best Position | Example Integration |
|---|---|---|
| Media mention | Headline (if short) or Description lead | "Dezeen Featured Design" (H) / "As featured in Dezeen and Architecture Australia." (D) |
| Certification | Description | "ATHA certified. Master Builder quality." |
| Statistic | Headline (number) or Primary text | "$74K/Yr Rental Income" (H) |
| Testimonial | Meta primary text (retargeting) | "'Best investment we've made' — Sarah K." |
| Award | Headline or Description | "Award-Winning Architecture" |
| User count / rating | Social Proof Stack (retargeting) | "500+ happy homeowners. 4.9 stars." |

### Source Label Rules
- Copy using hooks directly from creative brief → label `[BRIEF]`
- Copy you generate that doesn't exist in the brief → label `[GENERATED]`
- Copy adapted/modified from brief hooks → label `[ADAPTED]`
