# Sector Lens: SaaS & Software

Additional data points to extract when analyzing SaaS products and software businesses — web apps, mobile apps, developer tools, B2B platforms, freemium products.

Load this lens during Step 2 when the business type is SaaS/software/app/platform.

---

## Extra Extraction Points (Beyond Universal Framework)

### Product & Pricing Model
- Pricing tiers (free, starter, pro, enterprise) with feature breakdown
- Freemium vs free trial vs demo-only vs sales-led
- Trial duration and conversion mechanism
- Annual vs monthly pricing (and discount spread)
- Per-seat vs flat-rate vs usage-based pricing
- Enterprise/custom pricing (contact sales indicator)

### Platform & Distribution
- Web app, mobile app (iOS/Android), desktop, or multi-platform
- App store presence: ratings, review count, download estimates
- App store category and ranking
- Browser extension or API availability
- Integration ecosystem (Zapier, native integrations listed)

### Onboarding & Activation
- Sign-up flow complexity (email-only, SSO, credit card required)
- Onboarding sequence (product tour, setup wizard, empty states)
- Time-to-value indicators (how fast can user see benefit)
- Self-serve vs assisted onboarding
- Documentation quality (help center, API docs, tutorials)

### Growth & Retention Signals
- User count or customer logos displayed
- Case studies or success metrics shared
- G2/Capterra/ProductHunt presence and ratings
- Churn mitigation features (annual lock-in, switching cost)
- Community or user forum presence
- Product-led growth indicators (viral loops, referral programs, in-app sharing)

### Competitive Positioning
- Feature comparison pages (who they compare against)
- "Alternative to X" positioning
- Category they claim (what market they define themselves in)
- Key differentiator messaging (speed, price, simplicity, AI-powered)
- Open source vs proprietary

### Technical Signals
- Platform built on (if visible: Webflow marketing site, React app, etc.)
- Status page or uptime commitment
- Security certifications (SOC2, GDPR, ISO 27001)
- API-first positioning
- Data export or portability features

## Common BLANK Fields for SaaS

These frequently need client intake or product analytics access:
- Monthly active users (MAU)
- Trial-to-paid conversion rate
- Monthly/annual churn rate
- Average revenue per user (ARPU)
- Customer acquisition cost (CAC)
- Lifetime value (LTV)
- Feature adoption rates
- Free-to-paid funnel metrics
