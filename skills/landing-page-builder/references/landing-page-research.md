# Landing Page Research — Builder Rules

Load this file in Step 3 (page generation). These are data-backed construction rules, not evaluation criteria. The landing-page-audit handles evaluation.

Source: Perplexity Deep Research, April 2026. 99 references analyzed.

---

## 1. Page Architecture by Type

### 1.1 Retreat/Wellness Booking (High-Ticket $300-2000+)

**Length:** 10-14 sections, 1500-3000+ words. Long-form outperforms short by 37-220% for complex offers.

**Mandatory section order:**
1. **Hero** — outcome-focused headline, primary CTA ("Check Dates" / "Book Now"), 1 trust marker, hero image constrained to 40-50% viewport height on mobile
2. **Logistics Snapshot** — dates, location, price range ("investment from $X"), who it's for / not for
3. **Transformation Story** — emotional before/after narrative, why this retreat exists
4. **Program Overview** — structure (day-by-day or pillars), benefits-focused bullets
5. **What's Included** — schedule, inclusions vs exclusions, accommodation details
6. **Social Proof Block 1** — testimonials with photos, star ratings, outcome specifics
7. **Facilitator Credibility** — bios, credentials, media logos
8. **Pricing & Packages** — 3 tiers max (shared/private/VIP), anchor high-to-low, highlight recommended
9. **Social Proof Block 2** — case studies or transformation stories tied to outcomes
10. **FAQ / Objection Handling** — payment plans, cancellation, travel, dietary, safety
11. **Booking Form** — short or multi-step, deposit explanation
12. **Urgency / Scarcity** — limited spots, deadline, early bird (only if authentic)
13. **Travel & Practical Info** — getting there, packing list (optional)
14. **Final CTA + Summary** — reiterate promise, single primary CTA

### 1.2 Workshop/Event Registration (Mid-Ticket $50-300)

**Length:** 6-10 sections, 600-1500 words. Shorter pages suffice for simple local events.

**Mandatory section order:**
1. **Hero** — event name, date, location/online, 1-2 key benefits, "Register Now" above fold
2. **Key Details Strip** — date, time, location, duration, price (consider sticky sub-header)
3. **Value Proposition** — bullet list of outcomes linked to pains (burnout, flexibility, stress)
4. **Agenda/Schedule** — high-level breakdown (sessions, breaks, activities)
5. **Speaker/Teacher Profiles** — photos, credentials, prior events
6. **Social Proof** — past attendee testimonials, event photos, partner logos
7. **Pricing & Tickets** — early-bird vs standard, group discounts, what each includes
8. **FAQ** — logistics, refund policy, what to bring
9. **Registration Form or Button** — inline short form or anchor link to form
10. **Final CTA + Urgency** — limited seats, countdown (if real), last CTA

### 1.3 Lead Generation (Free Consultation/Enquiry)

**Length:** 4-7 sections, under 1000 words. Concise converts better for free offers.

**Mandatory section order:**
1. **Hero** — problem-solution headline, single CTA, short explainer subhead
2. **What You Get** — bullet list of consult outcomes (clarity, plan, diagnosis)
3. **Social Proof** — key testimonial, logos, simple stats
4. **Brief Credibility** — mini-bio, certifications, experience
5. **How It Works** — 3-step process (book, talk, plan)
6. **Short Form** — name + email (+ optional phone), or multi-step starting with yes/no question
7. **FAQ** — "Is this really free?", "Who is this for?", privacy reassurance

**Form rule:** 3-5 fields convert 25-120% better than 7+ fields.

### 1.4 Teacher Training/Certification ($2000+)

**Length:** 8-12 sections, 1500-3000+ words. Needs curriculum depth and accreditation proof.

**Mandatory section order:**
1. **Hero** — program name, credential, duration, cohort start date, "Apply"/"Download Curriculum"
2. **Who This Is For** — ideal candidate profile
3. **Outcome & Career Benefits** — what graduates can do, employment stats
4. **Program Structure** — phases or modules, visualized (zig-zag or scroll-linked)
5. **Curriculum Detail** — modules with outcomes, assessment, mentoring
6. **Accreditation** — Yoga Alliance or equivalent, partner institutions
7. **Learning Experience** — methods, cohort size, support, community
8. **Social Proof** — student testimonials, graduation stories, learner count
9. **Pricing & Payment** — tuition, payment plans, early-bird, scholarships, 3 tiers with anchoring
10. **FAQ & Policies** — admission, workload, attendance, refund
11. **Application Form** — multi-step intake to pre-qualify
12. **Final CTA + Summary** — reassurance + CTA

## 2. Above-the-Fold Rules (Mobile 375px)

These are non-negotiable for every page type:

1. **Headline + CTA visible without scrolling** — always
2. **Body text >= 16px, headline >= 24px** — no pinch-zooming
3. **Hero image constrained to 40-50% viewport height** — never push CTA below fold
4. **Include at least 1 trust signal** — badge, testimonial snippet, or count ("500+ guests hosted")
5. **Message match between ad and headline** — improves conversions 20-35%
6. **Subheading explains the outcome** — 1 sentence max

---

## 3. CTA Placement Rules

| Pattern | Data | When to Use |
|---|---|---|
| Sticky bottom bar (mobile) | +23% avg mobile conversion lift, +25% sales in one study | Long pages, all page types |
| Inline after key sections | Standard best practice | Every page — after social proof, after features, at bottom |
| Floating button | +55% CTA clicks, +7% transactions in one test | Consider for event/booking pages |

**CTA frequency rule:** Place CTAs before the 50% scroll mark. Repeat after every 2-3 sections.

**Sticky CTA caution:** Test for annoyance. For wellness funnels, use inline CTAs + subtle sticky bottom bar.

---

## 4. Social Proof Placement

**Distribute, don't isolate.** Social proof near claims and decisions, not in one testimonial wall.

| Position | What to Place |
|---|---|
| Near hero | Badge, count ("500+ guests hosted"), star rating |
| Beside benefit sections | Inline testimonial snippets |
| Near pricing/checkout | Outcome-focused testimonials, guarantee |
| Throughout page | 2-3 proof touchpoints minimum |

**Real-time notifications** can boost conversions up to 98% but risk annoyance. Use judiciously.

**Inline > popup** — embedded social proof is less disruptive and more trustworthy than notification popups.

---

## 5. Pricing Presentation Rules

1. **3 tiers maximum** — conversion drops ~17% moving from 3 to 5+ options
2. **Highlight recommended middle option** — visual emphasis (border, badge, scale)
3. **Anchor high-to-low** — show premium first, then mid, then basic. One experiment: 500% uplift in pricing CTA clicks with reversed order
4. **Show value before price** — for retreats, display comparable hotel+wellness costs before retreat price
5. **Crossed-out "regular" price** works when authentic
6. **Feature comparison table** — useful but hide behind toggle for complex offers

---

## 6. FAQ Rules

- Place near decision points (after pricing, before form) — reduces uncertainty
- Write to handle objections, not just inform: "What if I need to cancel?" > "What is your refund policy?"
- Keep answers concise — overly long FAQs introduce new doubts
- 5-8 questions is optimal for most page types
- A/B test FAQ presence — it can backfire if poorly written

---

## 7. Form Rules

| Principle | Data |
|---|---|
| 3-5 fields | 25-120% more completions than 7+ fields |
| Multi-step > single long form | Up to 23-45% improvement with progress indicators |
| Start with easy question | Yes/no or preference question first builds micro-commitment |
| Inline > modal | Embedded forms better for simple offers; modals OK when CTA-triggered |
| Click-to-call option | Doubles inbound conversion (30% to 66.7%) for service businesses |
| Input types matter | Use tel, email, date pickers on mobile for autofill |

**High-ticket forms:** Multi-step intake at top/mid-page ("Apply"), simpler CTA repeated lower.

**Lead gen forms:** Email-only often wins where qualification can happen later.

## 8. Typography System

### Rules:
- **Body text:** minimum 16px mobile, 16-18px desktop
- **Line height:** 1.4-1.6 for body
- **Line length:** 50-75 characters per line (desktop)
- **Max 2 font families** — display/heading + body
- **Heading hierarchy:** H1 > H2 > H3 with consistent sizing steps

### Wellness/Retreat Pairing Pattern:
- Bold display or serif headline (aspirational, premium feel)
- Clean sans-serif body (readable, modern)
- Mix weights: thin + heavy creates visual contrast

### Technical:
- Use variable fonts for fluid type scales across viewports
- `font-display: swap` to avoid invisible text during load
- Preload key font files
- Define system font fallbacks to minimize CLS on swap

---

## 9. Spacing & Layout System

### Spacing:
- **8px grid base** — all spacing multiples of 8
- **Section padding:** 48-80px vertical rhythm between sections
- **Mobile margins:** minimum 16px horizontal (never edge-to-edge text)
- **Consistent spacing** — same padding system throughout

### Layout Patterns (by effectiveness):
1. **Full-bleed alternating sections** — staple for retreat/course pages, storytelling. ALWAYS use.
2. **Bento grids** — modular tiles for features, testimonials, media. Good for engagement. Use selectively.
3. **Single-column stacked** — mobile default. Clear and scannable.
4. **Asymmetric/overlapping** — draws attention but must maintain reading order. Use for hero or 1-2 accent sections max.
5. **Horizontal scroll** — testimonial carousels, image galleries only. Never for essential copy.

### Simplification Impact:
- Removing clutter and simplifying layout yields 20-40% conversion improvement

## 10. Color Rules

### Wellness/Retreat Defaults:
- **Light backgrounds with ample whitespace** — conveys openness and calm (dark themes are for tech/SaaS)
- Calm, desaturated palette + 1-2 saturated accent colors for CTAs
- CTA button should have highest contrast ratio on page
- Gradients: restrained, in backgrounds or CTA buttons only

### Contrast Requirements:
- WCAG AA minimum: 4.5:1 for normal text, 3:1 for large text
- CTA color distinct from page palette — it must pop
- No information conveyed by color alone

### Card/Container Design:
- Simple high-contrast cards with clear borders or shadows
- Glassmorphism: sparingly, risks contrast/readability issues
- Neumorphism: avoid for conversion-critical controls

---

## 11. Image Rules

- **Hero image:** shows product, service, or outcome — never abstract stock
- **Rounded corners + soft shadows** — standard for wellness/retreat feel
- **Compression:** target median < 911KB total images (42% of page weight)
- **Responsive:** use `srcset` and `sizes` for different viewports
- **Lazy load:** `loading="lazy"` for all below-fold images
- **Format:** WebP (universal support) or AVIF (smaller but less support)
- **Parallax:** sparingly, GPU-accelerated only (transform + opacity), test on mid-range Android

## 12. Animation Rules

### What Helps Conversion:

| Animation | Impact | Implementation |
|---|---|---|
| Scroll-triggered reveals (fade-up, slide-in) | Guides attention, improves engagement | IntersectionObserver or CSS scroll-driven |
| Number counters for social proof | Draws attention to trust metrics | requestAnimationFrame + IntersectionObserver |
| Button hover micro-interactions (scale, color, shadow) | Increases perceived clickability | CSS transitions on :hover |
| Multi-step form progress indicators | 23-45% improvement in completion | CSS width animation on progress bar |
| Sticky CTA entrance animation | Draws attention without jarring | CSS transform slide-up on scroll threshold |

### What Hurts:
- Heavy parallax on low-end devices → higher bounce rates
- Animating layout properties (top, left, width, height, margin, padding) → triggers reflow
- Too many distinct animation sequences → performance degradation

### Performance Rules:
1. **Only animate `transform` and `opacity`** — GPU-accelerated, no reflow
2. **Use IntersectionObserver** for scroll triggers — not scroll event listeners
3. **Reuse consistent patterns** — same fade-up for all cards, not unique animations per element
4. **Test on mid-range Android + 4G** — 30% of devices are low-end
5. **Respect `prefers-reduced-motion`** — disable/simplify animations via media query (WCAG requirement)
6. **Limit complex animations to key sections** — hero, social proof counters, CTA. Not every element.

### Modern CSS (2026 Production-Ready):
- **CSS scroll-driven animations** — Chrome 115+, Safari 26+. Progressive enhancement with fallback.
- **View Transitions API** — Chrome 111+, Safari adding, Firefox late 2025. SPA-like transitions.
- **@starting-style** — entry animations without "jump". Modern browser support.
- **Anchor positioning** — emerging, check browser support before using.

**For Lovable rebuild:** CSS scroll-driven animations and view transitions can be progressively enhanced. Stick to IntersectionObserver + CSS transforms as the safe baseline.

## 13. Mobile-Specific Rules

### Thumb Zone:
- 75% of smartphone touches are thumb
- **Primary CTAs in bottom-center** ("green zone")
- Sticky bottom bars for main CTA

### Tap Targets:
- Minimum 44x44 CSS pixels (WCAG AAA + UX research standard)
- Adequate spacing between targets to prevent mis-taps

### Mobile Form:
- Input types: `type="email"`, `type="tel"`, date pickers
- Enable autofill
- 3-5 fields max, optional phone if not essential
- Full-width CTA button on mobile

### Scroll Depth:
- 57% of viewing time is above the fold
- Place main CTAs before 50% scroll mark
- Long-form content: target 70-80% scroll depth for key sections
- For short pages: 25-50% scroll depth acceptable if conversion is high

## 14. Conversion Psychology Rules

### Anchoring:
- Show high price first, then mid, then low
- Display comparable costs before your price ("5 nights at a resort: $2,400. This retreat: $799")
- Crossed-out "was" prices when authentic

### Paradox of Choice:
- 3 options optimal (6 vs 24 options = 30% vs 3% purchase rate in classic experiment)
- Never more than 4 pricing tiers
- One "recommended" option always highlighted

### Loss Aversion:
- "Only 4 spots left" + "Join 500+ others" — combines scarcity + social proof
- Frame positively for wellness: "Don't miss your chance to reset before Q4"
- Only use when scarcity is authentic

### Serial Position Effect:
- Put strongest benefits first and last in lists
- Middle items get forgotten — put supporting details there

### Micro-Commitments:
- Start forms with easy yes/no question before asking for details (23% conversion increase)
- Quizzes or self-assessment before consultation bookings increase perceived value
- Each "yes" makes the next "yes" easier

## 15. Performance Budget

| Metric | Target |
|---|---|
| LCP (Largest Contentful Paint) | <= 2.5 seconds |
| INP (Interaction to Next Paint) | <= 200ms |
| CLS (Cumulative Layout Shift) | <= 0.1 |
| Total page weight | < 1.5-2 MB (HTML + CSS + JS + images) |
| Image weight | < 40-45% of total |
| 3-second load on 4G | 53% of mobile users abandon if slower |

### Key Performance Rules:
1. Inline critical CSS for above-fold content
2. Defer non-critical JS
3. Lazy load below-fold images
4. Minimize third-party scripts (chat, tracking, social proof)
5. Use WebP/AVIF for images
6. Preload key fonts

---

## 16. Lovable/React Portability Rules

### DO:
- Semantic HTML wrappers (`section`, `header`, `main`, `footer`, `article`)
- Predictable class naming (BEM or Tailwind-style utilities)
- Design in repeated component blocks (hero, feature grid, testimonial slider, pricing cards)
- Flexbox/Grid for layout — clear structure
- Consistent spacing tokens that map to design system values

### DON'T:
- Canvas-based or complex SVG effects (hard to rebuild)
- Deeply nested wrappers only for animation
- Tight coupling between layout and JS calculations
- Complex CSS cascades that are hard to reverse engineer
- Heavy inline styles

### CSS Approach:
- Tailwind-style utility classes preferred — each class = one styling decision, maps cleanly to Lovable controls
- If vanilla CSS, use consistent BEM naming
- Avoid complex cascades and specificity wars

---

## 17. Exit Intent Patterns

- **Desktop:** mouse-leave triggers overlay
- **Mobile:** back-button or aggressive scroll-up triggers
- **What works for wellness:** "Download retreat guide" or "Get free consultation" — aligns with consultative journey
- **What doesn't work:** generic discount popups — gets dismissed
- Personalized, value-adding overlays only
- Never auto-launch modals — CTA-triggered only
