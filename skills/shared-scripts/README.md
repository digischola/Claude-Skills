# shared-scripts/

Cross-skill utilities used by all dashboard-generating skills. Lives at the skills/ level so every skill can import from it without duplication.

## Scripts

- **generate_dashboard.py** — Takes an HTML template with `{{KEY|default}}` placeholders + a data dict + optional brand-config.json, produces final HTML. Works as CLI or Python module import.

## How skills should import

```python
import sys, os
sys.path.insert(0, os.path.join(os.path.dirname(__file__), '..', 'shared-scripts'))
from generate_dashboard import generate_dashboard
```

## CLI usage

```bash
python generate_dashboard.py template.html data.json [brand-config.json] [output.html]
```

## Changelog

- **2026-04-16** — Fixed critical CSS injection bug. `apply_brand_colors()` was replacing scalar placeholders (PRIMARY_COLOR, DARK_BG, etc.) but silently discarded the `:root { --primary: ... }` block generated by `build_brand_css()`. Any template using CSS custom properties (`var(--primary)`, `var(--dark)`) was rendered with the template's default colors instead of the client's brand. Fix: `apply_brand_colors()` now injects a `<style id="brand-tokens">` block immediately before `</head>` (case-insensitive match). Falls back to prepending if no `</head>` tag exists. Smoke-tested end-to-end; brand CSS variables now flow through correctly.
