# assets/ — minimal by design

Most personal-brand skills keep templates, fonts, and brand icons here. Visual-generator is different: its "assets" are **Remotion React components + Hyperframes HTML blocks + the brand wiki** — all of which are versioned in the user's brand repo, not duplicated in the skill.

## Where the actual production assets live

| Asset | Location |
|---|---|
| Remotion compositions + primitives | `Desktop/Digischola/brand/_engine/remotion-studio/src/` |
| Hyperframes catalog blocks | `Desktop/Digischola/brand/_engine/hyperframes-scenes/compositions/` |
| Shared design tokens (source of truth) | `Desktop/Digischola/brand/_engine/remotion-studio/src/theme/design-tokens.json` |
| Generated tokens.css | `Desktop/Digischola/brand/_engine/hyperframes-scenes/shared/tokens.css` (autogenerated) |
| Fonts | Google Fonts imported in tokens.css (Orbitron / Space Grotesk / Manrope) |
| Outro portrait | `Desktop/Digischola/brand/_engine/face-samples/face-01.jpg` |
| Rendered output | `Desktop/Digischola/brand/queue/assets/<entry_id>/` (queue/ stays at top — unchanged) |

## What CAN live here

If the skill ever needs:
- Brief templates (Path A Claude Design handoff) → `assets/brief-template.md`
- Static quote-card HTML templates → `assets/quote-card-template.html`
- JSON schema definitions → `assets/*.schema.json`
- Fonts we DON'T want to pull from Google Fonts (e.g. custom licensed) → `assets/fonts/`

For now, none of these apply. The skill is primarily a Python orchestrator; its "assets" are the compositions in the user's brand repo.
